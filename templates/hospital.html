Institutional Status

This institution is participating in the federated learning network.

Training Participation Procedure

The federated learning workflow operates as follows:

  1. The hospital performs local training using its internal dataset.
  2. Raw patient data remains within the hospital infrastructure.
  3. After local training, only model weights and performance metrics are generated.
  4. The hospital submits these artifacts to the coordination server.
  5. The aggregated global model is received for the next round.

The training process is executed locally within the hospital environment. Execution of the training procedure is performed through the provided Jupyter notebook (Medledger_rev2.ipynb). This notebook manages all aspects of local training, submission, and model updates.

Privacy and Data Protection

Patient data does not leave the hospital infrastructure. The server receives only model parameters and aggregated performance statistics. No patient-level information, demographic data, or clinical records are transmitted to the coordination server or to other institutions.

Monitoring and Status

The current training round and aggregation status can be viewed from the model status dashboard. This dashboard displays round numbers, hospital submission counts, and aggregation progress.
